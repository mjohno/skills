# Artifact: PRD

## Source References
- **PRD template**: `design/assets/prd_template.md`
- **User story template**: `design/assets/user_story_template.md`

## Target Structure
A PRD has these sections:
1. Executive Summary — concise overview of the product/feature
2. Strategic Goals — SMART table with detailed descriptions
3. User Stories — table with numbered detail (As a/ I want/ so that)
4. Acceptance Criteria — Gherkin scenarios (Given/When/Then)

## Target Metadata
- Typically named `PRD-NNN-<name>.md` or similar
- Lives in a design or product directory
- Often referenced by RFCs and implementation plans
