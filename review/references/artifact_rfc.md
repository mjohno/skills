# Artifact: RFC

## Source References
- **RFC skill**: `rfc/SKILL.md`

## Target Structure
An RFC has these sections:
1. Context — background and motivation
2. Proposed Solution — clear description of the approach
3. Design Decisions — rationale for key choices with trade-offs
4. Architecture — system design, diagrams, data flow
5. Feasibility — technical, schedule, resource assessment
6. Alignment with PRD — if applicable
7. Open Questions — actionable, time-bound
8. Decision Log — decisions with rationale and date

## Target Metadata
- Typically named `RFC-NNN-<name>.md` or similar
- Lives in a design or engineering directory
- Often references a PRD
