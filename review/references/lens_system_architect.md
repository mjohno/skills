# Lens: System Architect

Migrated from `design/references/workflow_rfc_review.md`.

## Criteria

### 1. System Boundaries
- Are system boundaries clearly defined?
- Are interfaces between systems well-specified?

### 2. Data Flow
- Is data flow clearly documented?
- Are data transformations and storage points identified?

### 3. API Contracts
- Are API contracts complete and well-defined?
- Are versioning and backward compatibility considered?

### 4. Scalability
- Does the design support the required scale?
- Are bottlenecks identified and addressed?

### 5. Technology Selection
- Are technology choices justified?
- Are alternatives considered?
