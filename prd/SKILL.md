# prd

## Goal
Capability to manage the lifecycle of Product Requirement Documents (PRDs), including Strategic Goals, User Stories, and Gherkin-formatted Acceptance Criteria.

## Use When
- You need to create a new PRD (see [Write Mode](references/mode_write.md)).
- You need to audit requirement quality (see [Review Mode](references/mode_review.md)).
- You need to verify PRD compliance (see [Validate Mode](references/mode_validate.md)).

## Workflows

### [Write Mode](references/mode_write.md)
For scaffolding new PRDs from high-level ideas or business goals.

### [Review Mode](references/mode_review.md)
For auditing existing PRDs for ambiguity, gaps, and logical inconsistencies.

### [Validate Mode](references/mode_validate.md)
For programmatically and semantically verifying that a PRD meets mandatory quality standards.
