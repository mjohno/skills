# Spec: Prose

This spec defines criteria for reviewing prose, documents, comments, and documentation text.

## Source References
- **Readability guidelines**: `../../writing/references/readability_guidelines.md`
- **Density guidelines**: `../../writing/references/density_guidelines.md`
- **Review mode**: `../../writing/references/mode_review.md`

## Additional Criteria

### Readability
1. Sentence length is appropriate (not too long, not too short)
2. Sentence length varies to prevent reader fatigue
3. Average sentence length is reasonable (not staccato, not dense)
4. Clear, concise, and easy to scan

### Density
1. Writing is information-dense (not heavy on function words)
2. No wordy phrases (e.g., "in order to" → "to")
3. Active voice preferred over passive
4. No redundant words

### Style
1. Human-sounding, not AI-generated or templated
2. No AI-sounding filler
3. Preserves user's voice when possible
4. No robotic or generic phrasing
