# UI Mockup Prototype Quality Checklist

Use this checklist when outlining, drafting, modifying, or reviewing a single-page HTML UI mockup prototype.

## Critical
- [ ] States the decision the mockup will inform.
- [ ] Names the user task or comprehension risk being tested.
- [ ] Identifies the target user and context.
- [ ] Represents the primary screen or flow at the minimum useful fidelity.
- [ ] Makes primary actions, labels, states, and hierarchy visible.
- [ ] Includes enough realistic content to test comprehension.
- [ ] Marks non-functional controls or simulated behavior.
- [ ] Defines validation signals, such as task completion, correct interpretation, or user confidence.
- [ ] Defines failure signals, such as misclicks, wrong expectations, or inability to explain the page.
- [ ] Lists omitted production concerns.

## Quality
- [ ] Fits in one self-contained HTML file.
- [ ] Avoids unnecessary styling complexity.
- [ ] Uses semantic HTML where practical.
- [ ] Includes notes or comments for what to observe during testing.
- [ ] Can be opened locally without build tools.
- [ ] Ends with a clear next decision.
