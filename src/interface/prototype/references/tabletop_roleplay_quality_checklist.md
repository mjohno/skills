# Tabletop / Roleplay Prototype Quality Checklist

Use this checklist when outlining, drafting, modifying, or reviewing a low-fidelity tabletop exercise or roleplay prototype.

## Critical
- [ ] States the decision the exercise will inform.
- [ ] Names the risky assumption being tested.
- [ ] Defines participant roles and who observes.
- [ ] Provides a realistic scenario trigger and starting context.
- [ ] Contains step-by-step facilitation instructions.
- [ ] Includes prompts that force decisions, handoffs, or tradeoffs.
- [ ] Defines observable validation signals.
- [ ] Defines failure signals and stop conditions.
- [ ] Captures debrief questions and evidence to record.
- [ ] Explicitly lists what is omitted or simulated.

## Quality
- [ ] Can be run in 30–60 minutes without extra preparation.
- [ ] Uses plain language suitable for participants.
- [ ] Avoids leading participants toward the desired answer.
- [ ] Includes at least one complication or edge case.
- [ ] Separates facilitator script from observer notes.
- [ ] Ends with a clear next decision.
