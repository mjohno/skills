# learn

Goal: Reduce memory entropy into stable lessons, summary updates, and compression decisions.
Non-Goals: Knowledge-base promotion, raw expansion, or direct file writes.
Use-When: A memory workflow needs deduplication, consolidation, or contradiction handling.

## 0. Prerequisites
- Retrieved or raw memory input
- The `interface/memory` contract

## 1. Inputs
- Memory themes, bullets, and candidate summary material
- Optional approval context for destructive summary changes

## 2. Processes
1. Read and follow the `interface/memory` contract.
2. Dedupe and consolidate messy memory into concise, stable summary updates.
3. Resolve or flag contradictions.
4. Identify stale or temporary entries.
5. Decide what should stay in the log versus what can be compressed into the summary.
6. Do not decide what should be promoted to the knowledge base; memory and knowledge base remain uncoupled.

## 3. Outputs
- Stable summary updates
- Compression candidates
- Contradiction flags or rewrite candidates

## 4. Next Steps
- `interface/memory` — format summary updates for writing
- `map/dream` — apply cleanup workflow
