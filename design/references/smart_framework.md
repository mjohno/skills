# S.M.A.R.T. Framework Reference

To ensure strategic goals are actionable and measurable, all goals defined in a PRD must satisfy the following five criteria:

## 1. Specific (S)
The goal must be clear and unambiguous. It should define exactly what needs to be achieved.
- **Bad**: "Improve the app."
- **Good**: "Reduce the checkout latency for mobile users."

## 2. Measurable (M)
There must be a concrete metric or way to track progress and confirm the goal has been met.
- **Bad**: "Make the app faster."
- **Good**: "Achieve a p95 response time of < 200ms for the checkout API."

## 3. Achievable (A)
The goal must be realistic given the current technical constraints, resources, and timeline.
- **Bad**: "Achieve 100% uptime with zero maintenance."
- **Good**: "Achieve 99.9% uptime for the production environment."

## 4. Relevant (R)
The goal must align with the broader business objectives and the core problem the product is solving.
- **Bad**: "Add a social sharing feature" (when the goal is purely performance-related).
- **Good**: "Improve conversion rates by reducing checkout friction."

## 5. Time-bound (T)
The goal must have a defined start and end point, or a specific target date/window.
- **Bad**: "Improve performance soon."
- **Good**: "Achieve the target latency by the end of Q3."
